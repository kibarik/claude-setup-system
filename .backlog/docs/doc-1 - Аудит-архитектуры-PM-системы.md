## Найденные противоречия (7)

1. **[CONTRADICTION-001] Дублирование CLAUDE.md** (high) — файл существовал в двух местах: корень проекта и .claude/ — идентичное содержимое, путаница с источником истины. ❌ ИСПРАВЛЕНО

2. **[CONTRADICTION-002] PM не выполняет Bash vs PM использует Bash** (high) — в жёстких запретах указано что PM не выполняет Bash, но в Шаге 0 PM выполняет множество Bash команд для проверки инструментов. ❌ ИСПРАВЛЕНО

3. **[CONTRADICTION-003] Проверка MCP до vs после копирования .claude** (critical) — таблица запрещает проверять MCP до копирования .claude, но алгоритм инициализации требует проверки MCP первым. ❌ ИСПРАВЛЕНО

4. **[CONTRADICTION-004] SA запрещает Write() файлов спецификаций** (medium) — SA запрещает Write(), но использует Spec-Kitty который создаёт файлы. ❌ ИСПРАВЛЕНО

5. **[CONTRADICTION-005] PM не может vs PM делегирует** (medium) — PM должен делегировать всё что выходит за рамки, но сам устанавливает MCP инструменты. ❌ ИСПРАВЛЕНО

6. **[CONTRADICTION-006] Механизм связи TASK_ID <-> REVIEW_TASK_ID** (high) — PM использует псевдокод `find_review_task_for()` который не определён. Связь может быть потеряна. ❌ ИСПРАВЛЕНО

7. **[CONTRADICTION-007] QA не исправляет баги vs создаёт баг-задачи** (low) — QA не должен исправлять, но создаёт баг-задачи. Правильное поведение, но может быть сформулировано яснее. ✅ ОК
