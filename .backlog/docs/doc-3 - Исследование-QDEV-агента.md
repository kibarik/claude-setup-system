# Исследование задачи TASK-14

## Контекст и понимание задачи

Необходимо создать QDEV-агента (Quality DEV Verification) который проверяет, что код действительно запускается и работает, между DEV и REVIEW этапами.

## Проблема

DEV-агент может сообщать "сервис запущен" или "тесты пройдены", но на практике ничего не работает. Агент может непреднамеренно вводить в заблуждение или недостаточно проверять результаты.

## Изученные артефакты

1. `.claude/agents/reviewer.md` — REVIEW агент, который проверяет качество кода
2. `.claude/agents/developer.md` — DEV агент, который пишет код
3. `.claude/agents/qa.md` — QA агент для финального тестирования
4. `CLAUDE.md` — PM система с Фазой 3 (параллельный оркестратор)
5. `doc-2 - TIMEOUTS-для-агентов.md` — информация о timeout'ах агентов

## Ключевые технические решения

[DEV-DECISION] QDEV проверяет факты запуска, не качество кода | обоснование: REVIEW занимается качеством кода, QDEV должен быть быстрым и проверять только запуск | альтернативы: расширить REVIEW (слишком долго), оставить как есть (проблема остаётся)

[DEV-DECISION] QDEV работает в Фазе 3 между DEV и REVIEW | обоснование: PM уже запускает агентов параллельно, добавить ещё одну очередь | альтернативы: отдельная фаза (разрывает поток), часть DEV (агент не проверяет себя объективно)

[DEV-DECISION] QDEV timeout = 10 минут | обоснование: быстрый агент, только проверка фактов | альтернативы: 5 минут (мало для запуска сервисов), 15 минут (слишком долго)

## Выбранный подход

1. Создать файл `.claude/agents/qdev.md` с описанием агента
2. Добавить новый статус `qdev-check` в backlog config
3. Интегрировать QDEV в Фазу 3 оркестратора:
   - После DEV завершает, статус -> qdev-check
   - QDEV проверяет запускаемость
   - При PASS -> code-review
   - При FAIL -> review-debug (возврат к DEV)
4. Добавить QDEV timeout в TIMEOUTS словарь

## Риски и edge cases

- QDEV может быть избыточен для простых задач (добавить флаг skip для тривиальных изменений)
- Запуск сервисов может требовать дополнительные ресурсы (QDEV должен быть лёгким)
- Ложные срабатывания (сервис не запустился по временной причине) — добавить retry
