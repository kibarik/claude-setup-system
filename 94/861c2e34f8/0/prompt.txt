Добавь в беклог задачу для "микросервисная оркестрация"

ASIS:
Мы работаем в рамках одного монорепозитория

ПРОБЛЕМА:
Реальные проекты построены на микросервисах и требуют более сложную систему взаимодействий, коммуникации между агентами

TOBE:
Модель управления через PM, Техлидов, Архитекторов, управление состояниями команд, отслеживанием исполнения между разными агентами

Хочу продумать конфигурацию для микросервисной архитектуры с Git Submodule. PM-агент в этой системе выступает координатором всех действий. Распределяет задачи на техлида нужного домена. Техлид в своем домене уже следит за исполнением зоны. PM выступает координатором между микросервисами и хранителем общего состояния и беклога, техлид хранит свой собственный беклог микросервсиа. 

Основная архитектура:
/ - корневая папка с PM-агентов и конфигурацией
/infra - репозиторий с инфраструктурой для запуска
/docs - большая библиотека репозитория, здесь хранятся описание фичей продукта, ведется вся документация
/srv-webui - frontend приложение на nextJS
/srv-orchestrator - сервис backend оркестрации
/srv-worker-{name} - отдельный микросервис исполнителя, вызывается из orchestrator для исполнения поставленной задачей. Таких воркеров может быть множество, это все микросервисы

Взаимодействие между всеми микросервисами идет через общий Temporal и Supabase. В отдельности каждый микросервис при необходимости может иметь отдельную локальную минибазу для исполнения обязанностей, но все ключевые события должны управляться централизованно.

Взаимодействие происходит по принципу:
- PM получает задачу от человека
- PM собирает требования с помощью собственного аналитика
- PM после завершения сбора передает требования архитектору (ARC)
- ARC после изучения требования на основе имеющейся информации разделяет задачу что от каждого микросервиса он ожидает.
- ARC согласовывает свои требования с системный аналитиком
- ARC передает задачу PM на исполнение
- PM 


 в каждый микросервис и инициирует у техлида запрос информации, входит ли какая-то часть в зону ответственности этого микросервиса
- Техлид микросервиса
