# Task Completion Checklist: claude-setup

## При завершении любой задачи

### Для SA (Analyst)
- [ ] `[SA-REPORT]` записан в notes задачи
- [ ] Все 4 этапа Spec-Kitty пройдены
- [ ] Подзадачи созданы через backlog__task_create()
- [ ] У каждой подзадачи есть PASS/FAIL критерий
- [ ] У каждой подзадачи есть сценарий демонстрации
- [ ] Зависимости (depends_on) проставлены
- [ ] `[SA-ASSUMPTION]` задокументированы (если есть)
- [ ] `/spec-kitty dashboard` показывает все 4 раздела заполненными

### Для Scrum Master
- [ ] `[SCRUM-REPORT]` записан в notes эпика
- [ ] Все подзадачи прошли Token Budget Gate (≤175k токенов)
- [ ] Каждая задача самодостаточна (без внешних ссылок)
- [ ] Зависимости проставлены для всего бэклога
- [ ] PM уведомлён и подтвердил готовность к разработке

### Для Developer
- [ ] `[DEV-LOG]` записан в notes задачи
- [ ] Задача реализована полностью
- [ ] PR открыт и ссылка зафиксирована в evidence
- [ ] Ветка/Worktree зафиксированы в DEV-LOG
- [ ] Задача переведена в статус `code-review`
- [ ] Самопроверка пройдена

### For Reviewer
- [ ] `[REVIEW-REPORT]` записан в notes review-задачи
- [ ] Код соответствует требованиям задачи
- [ ] PASS/FAIL критерий проверен
- [ ] Сценарий демонстрации проверен
- [ ] Вердикт зафиксирован (approve/request changes)
- [ ] Задача обновлена:
  - → `ready-for-testing` при одобрении
  - → `review-debug` при отклонении

### Для QA
- [ ] `[QA-LOG]` записан в notes задачи
- [ ] Тесты запущены (API + Playwright если применимо)
- [ ] Evidence содержит реальный вывод тестов (не слова)
- [ ] Вердикт PASS или FAIL с конкретикой
- [ ] При FAIL — баги задокументированы (шаги + ожидаемое + факт)

### Для PM (финальная проверка)
- [ ] Все задачи: done или cancelled
- [ ] Каждый переход статуса имеет `[PM-LOG]` с evidence
- [ ] Нет задач с depends_on на незакрытые задачи
- [ ] E2E тесты зелёные (если применимо)
- [ ] QA Gate пройден

## Evidence Requirements (жёстко)

**✓ Valid evidence:**
- `task_id=123`
- `file=/path/to/file.md`
- `https://github.com/...`
- `commit=abc123def`
- `<вывод команды тестов>`

**✗ Invalid evidence (слова, не данные):**
- `проверено`
- `выполнено`
- `готово`
- `ок`

## Critical Checkpoints

### Before Phase 3 (Development)
- [ ] CHECKPOINT пройден — человек явно сказал "да"/"go"/"запускай"
- [ ] Все задачи из SA прошли Scrum верификацию
- [ ] Token Budget Gate пройден для всех задач

### Before Phase 5 (QA Gate)
- [ ] Code review пройден (статус ready-for-testing)
- [ ] QA вынес вердикт PASS с evidence
- [ ] Сценарий демонстрации проверен

### Before Close (Done)
- [ ] QA Gate пройден
- [ ] Нет незакрытых зависимостей
- [ ] Все переходы задокументированы
